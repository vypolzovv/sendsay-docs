# Как настроить сегменты по данным продаж

С Модулем **Продажи** вы можете сегментировать подписчиков по тем данным, которые передаются с&nbsp;сайта в&nbsp;систему:

- версия Модуля &laquo;Продажи Старт&raquo; позволяет создавать сегменты на&nbsp;основе клиентских данных только за&nbsp;последние 48&nbsp;часов. Для условий недоступны события: Поиск, Добавление в&nbsp;избранное, Предварительный заказ, Товар появился, Изменение цены,
- с&nbsp;полной версией Модуля &laquo;Продажи&raquo; вы&nbsp;можете создавать сегменты без ограничений.

  [Сравнение версий Модуля Продажи](https://docs.sendsay.ru/ecom/how-to-connect-ecom/#возможности-версий)

:::tip Важно
Настроить сегменты можно только по&nbsp;клиентским событиям и&nbsp;данным, которые передаются с&nbsp;сайта в&nbsp;Sendsay. Например, если настроена передача события **Просмотр товара**, но&nbsp;в&nbsp;событии передаются лишь id&nbsp;товаров без их&nbsp;производителей, то&nbsp;сегментировать аудиторию по&nbsp;просмотренным товарам конкретного производителя не&nbsp;получится.

[Как настроить передачу данных с&nbsp;сайта](https://docs.sendsay.ru/ecom/how-to-configure-data-transfer/#настройка-модуля)
:::

Чтобы настроить условия для сегментов по&nbsp;данным продаж, перейдите в&nbsp;раздел [**Подписчики &rarr; Сегменты**](https://app.sendsay.ru/subscribers/segments) и&nbsp;выберите пункт **Продажи** в&nbsp;качестве условия.

![How to create a segment](/img/ecom/segmentation-by-ecom-data/how-to-create-segment.gif)

[Как создать сегмент ](https://docs.sendsay.ru/subscribers/lists-and-segments/what-is-segment/#как-создать-сегмент)

## Фильтр условий клиентских событий

В&nbsp;настройках сегмента можно уточнить условия клиентских событий с&nbsp;помощью кнопки с&nbsp;изображением фильтра:

![Event filter](/img/ecom/segmentation-by-ecom-data/event-filter.png)

- **Количество**&nbsp;&mdash; определяет количество действий, совершённых посетителем сайта. Например, следующее условие будет говорить о&nbsp;том, что в&nbsp;сегмент попадут те&nbsp;подписчики, которые оформляли заказ больше трёх раз:

  ![Over three orders](/img/ecom/segmentation-by-ecom-data/over-three-orders.png)

- **Время**&nbsp;&mdash; дата события. Например, если событие произошло сегодня или за&nbsp;какой-то конкретный период:

  ![Event runtime](/img/ecom/segmentation-by-ecom-data/event-runtime.png)

- **Атрибуция**&nbsp;&mdash; позволяет отследить события, которые произошли после взаимодействия получателя с&nbsp;рассылкой. Здесь есть четыре группы выпусков:

  ![Attribution](/img/ecom/segmentation-by-ecom-data/attribution.png)

  - `<любой выпуск>` &mdash; подписчики попадут в&nbsp;сегмент, если совершат указанное событие после перехода на&nbsp;сайт из&nbsp;любой рассылки, отправленной в&nbsp;любой или конкретный период,
  - `<любой выпуск по группе>` &mdash; подписчики попадут в&nbsp;сегмент, если совершат указанное событие после перехода на&nbsp;сайт из&nbsp;любого выпуска, отправленного по&nbsp;определённому сегменту в&nbsp;любой или конкретный период,
  - `<любой выпуск по шаблону>` &mdash; подписчики попадут в&nbsp;сегмент, если совершат указанное событие после перехода на&nbsp;сайт из&nbsp;выпуска, отправленного по&nbsp;конкретному шаблону в&nbsp;любой или конкретный период,
  - `название выпуска` &mdash; подписчики попадут в&nbsp;сегмент, если совершат указанное событие после перехода на&nbsp;сайт из&nbsp;конкретного отправленного выпуска.

- **Товары**&nbsp;&mdash; фильтр отбирает товар по&nbsp;его параметрам, передаваемым в&nbsp;Sendsay. Например, при следующих выставленных условиях в&nbsp;сегмент попадут те&nbsp;подписчики, которые просматривали на&nbsp;сайте рюкзаки:

  ![Product name](/img/ecom/segmentation-by-ecom-data/product-name.png)

  Если&nbsp;же в&nbsp;качестве условия добавить второй параметр&nbsp;&mdash; например, цену,&nbsp;&mdash; то&nbsp;в&nbsp;сегмент попадут подписчики, которые просматривали рюкзаки от&nbsp;5000&nbsp;рублей:

  ![Product name and price](/img/ecom/segmentation-by-ecom-data/product-name-and-price.png)

- **Свойства**&nbsp;&mdash; фильтрует значения дополнительных полей, передаваемых с&nbsp;сайта в&nbsp;Sendsay. У&nbsp;событий **Сделал заказ** и&nbsp;**Обновлял корзину** в&nbsp;качестве фильтра можно также выбрать данные транзакции&nbsp;&mdash; например, её&nbsp;ID или сумму заказа:

  ![Order amount](/img/ecom/segmentation-by-ecom-data/order-amount.png)

  В&nbsp;этот сегмент попадут те&nbsp;подписчики, которые сделали заказ на&nbsp;сумму 10000 рублей или больше.

  При выборе условия **Статус транзакции** необходимо использовать ID&nbsp;статусов, описанных в&nbsp;[таблице статусов заказов](https://docs.sendsay.ru/ecom/how-to-configure-data-transfer/#статусы-заказа). Например, если в&nbsp;сегмент должны попасть подписчики, оплатившие заказ, условия будут выглядеть так:

  ![Transaction status](/img/ecom/segmentation-by-ecom-data/transaction-status.png)

Настраивая сегмент, можно выбрать один или несколько фильтров для уточнения условий. Например, при следующих выставленных условиях в&nbsp;сегмент попадут подписчики, которые не&nbsp;меньше двух раз просматривали рюкзаки стоимостью от&nbsp;5000 рублей за&nbsp;последние 7&nbsp;дней:

![Several filters](/img/ecom/segmentation-by-ecom-data/several-filters.png)

## Примеры условий для сегментов по&nbsp;данным продаж

### Сегмент подписчиков, которые оформили заказ после перехода из&nbsp;конкретного письма

Например, необходимо выделить в&nbsp;сегмент подписчиков, кто приобрёл телевизор, перейдя на&nbsp;сайт из&nbsp;конкретного выпуска,&nbsp;&mdash; чтобы отправить по&nbsp;этому сегменту письмо с&nbsp;сопутствующими товарами. Тогда условия будут такими:

```
Продажи → Сделал заказ
Атрибуция → Название отправленного выпуска
Товары → Название продукта → содержит → Телевизор
```

![Example](/img/ecom/segmentation-by-ecom-data/example1.png)

Если необходимо выделить подписчиков, которые после перехода из&nbsp;рассылки купили телевизоры стоимостью, например, больше 20000&nbsp;рублей, к&nbsp;условиям товара необходимо добавить:

```
Цена → больше или равно → 20000
```

![Example](/img/ecom/segmentation-by-ecom-data/example2.png)

### Сегмент подписчиков, которые просматривали товары конкретного производителя

Например, вы&nbsp;хотите отправить проморассылку со&nbsp;скидкой на&nbsp;мебель производителя &laquo;Белый город&raquo; по&nbsp;тем подписчикам, кто просматривал товары конкретно этого производителя. Условия сегмента будут такими:

```
Продажи → Просматривал товар
Товары → Производитель → равно → Белый город
```

![Example](/img/ecom/segmentation-by-ecom-data/example3.png)

В&nbsp;случае если необходимо выделить подписчиков, которые просматривали товары этого производителя за&nbsp;конкретный промежуток времени, к&nbsp;условиям необходимо добавить фильтр **Время**:

```
Время → за 7 дней
```

![Example](/img/ecom/segmentation-by-ecom-data/example4.png)

### Сегмент подписчиков, которые наиболее часто просматривали товары конкретной категории

Например, необходимо выделить наиболее заинтересованных подписчиков, которые несколько раз просматривали категорию &laquo;Беспроводные наушники&raquo;,&nbsp;&mdash; чтобы оповестить их&nbsp;об&nbsp;акции на&nbsp;эту категорию товаров. Тогда условия для сегмента будут такими:

```
Продажи → Просматривал категорию
Количество → больше → 2 раз
Товары → Категория → равно → Беспроводные наушники
```

![Example](/img/ecom/segmentation-by-ecom-data/example5.png)

### Сегмент подписчиков, которые сделали минимум&nbsp;X заказов за&nbsp;последний месяц

Например, необходимо выделить наиболее лояльных клиентов&nbsp;&mdash; и&nbsp;рассказать им&nbsp;в&nbsp;рассылке о&nbsp;том, что они стали участниками программы лояльности. Условия для такого сегмента:

```
Продажи → Сделал заказ
Количество → не менее → X раз
Время → за 30 дней
```

![Example](/img/ecom/segmentation-by-ecom-data/example6.png)

Если необходимо, чтобы каждый из&nbsp;этих заказов был на&nbsp;сумму не&nbsp;менее&nbsp;N рублей, необходимо добавить условие:

```
Свойства → Сумма → больше или равно → N
```

![Example](/img/ecom/segmentation-by-ecom-data/example7.png)

### Сегмент подписчиков, которые положили в&nbsp;корзину товар стоимостью больше определённой суммы

Например, необходимо выделить в&nbsp;сегмент тех подписчиков, кто положил в&nbsp;корзину дорогостоящий товар и&nbsp;ушёл. Таких пользователей можно подтолкнуть к&nbsp;покупке, предложив им&nbsp;скидку, бесплатную доставку или другой бонус. Условия для такого сегмента:

```
Продажи → Обновлял корзину
Товары → Цена → больше или равно → N
```

![Example](/img/ecom/segmentation-by-ecom-data/example8.png)

В&nbsp;случае если необходимо конкретизировать товар, можно добавить в&nbsp;условия его название или другие параметры:

```
Товары → Название продукта → содержит → название
```

![Example](/img/ecom/segmentation-by-ecom-data/example9.png)

Или, например, период, в&nbsp;течение которого пользователи откладывали в&nbsp;корзину такой товар:

```
Время → в период → диапазон даты и времени
```

![Example](/img/ecom/segmentation-by-ecom-data/example10.png)

### Сегмент подписчиков, которые купили конкретный товар месяц назад

Например, необходимо выделить в&nbsp;сегмент пользователей, которые совершили покупку месяц назад&nbsp;&mdash; с&nbsp;целью напомнить им&nbsp;о&nbsp;том, что пора пополнить запасы. Условия для такого сегмента:

```
Продажи → Сделал заказ
Товар → Название продукта → содержит → название
Время → за интервал → от Y до W дней назад
```

![Example](/img/ecom/segmentation-by-ecom-data/example11.png)
